## 11.4 Latent MAS（潜在空间转移）

### 11.4.1 核心实现

**位置**：`methods/latent_mas_hybird.py:102`

```python
class LatentMASMethod:
    def __init__(self, model, agent_models=None, latent_steps=10, ...):
        self.latent_steps = latent_steps
        self.method_name = 'latent_mas_hybrid'
```

### 11.4.2 通信流程

```
Planner:
  问题 + 隐藏状态 h_A
          │
          ▼
  transfer_via_realignment(h_A, Model A, Model B)
          │
          ▼
  潜在向量传递给 Critic

Critic:
  接收潜在向量，继续推理...
```

### 11.4.3 核心：跨模型对齐

```python
# latent_mas_hybird.py:18-99
def transfer_via_realignment(
    hidden_states: torch.Tensor,
    model_from: ModelWrapper,
    model_to: ModelWrapper,
    lambda_reg: float = 1e-5
) -> torch.Tensor:
    """
    跨模型潜在空间对齐

    W_cross = (W_out_A^T @ W_out_A + λI)^-1 @ W_out_A^T @ W_in_B
    embedding_B = hidden_A @ W_cross
    """
    # 获取权重
    W_out_A = model_from.model.get_output_embeddings().weight
    W_in_B = model_to.model.get_input_embeddings().weight

    # 计算对齐矩阵
    gram = torch.matmul(W_out_A_f32.T, W_out_A_f32)
    gram_reg = gram + lambda_reg * I
    rhs = torch.matmul(W_out_A_f32.T, W_in_B_f32)
    W_cross = torch.linalg.solve(gram_reg, rhs)

    # 应用对齐
    embeddings_B = hidden_states @ W_cross
    return embeddings_B
```

### 11.4.4 特点

| 方面 | 说明 |
|------|------|
| **通信介质** | 连续潜在向量 |
| **信息损失** | 低（无量化） |
| **延迟** | 低（无文本生成） |
| **异构支持** | 受限（需要对齐矩阵） |
| **实现复杂度** | 中 |

### 11.4.5 局限性

1. **需要对齐**：不同模型需要训练对齐矩阵
2. **同构假设**：假设两个模型的 embedding 空间可对齐
3. **词表依赖**：效果依赖于词表重叠度

## 11.5 Vision Latent MAS Codec New（Vision Wormhole）

### 11.5.1 核心实现

**位置**：`methods/vision_latent_mas_codec_new.py:1025`

```python
class VisionLatentMASMethodCODECNew:
    def __init__(self, args, models):
        # 加载编解码器

        self.decoders: List[UniversalToVisionDecoder] = []

        # 加载检查点

```

### 11.5.2 通信流程

```
Agent A (Sender):
  h_A [B, L, H_A]
      │
      ▼
  Encoder_A(h_A) → U [B, K+2, d_univ]
      │
      │ 传输
      ▼
Agent B (Receiver):
  U [B, K+2, d_univ]
      │
      ▼
  Decoder_B(U) → delta [B, K_img, H_B], gate [B, 1, 1]
      │
      ▼
  vision_tokens = dummy + gate * delta
      │
      ▼
  VLM B 接收（通过视觉通道）
```

### 11.5.3 特点

| 方面 | 说明 |
|------|------|
| **通信介质** | 视觉通道（通用视觉空间） |
| **信息损失** | 低 |
| **延迟** | 低 |
| **异构支持** | 完全（通过编解码器） |
| **实现复杂度** | 高（需要训练编解码器） |

### 11.5.4 核心优势

1. **模型无关**：任何 VLM 都可以接入
2. **文本无关**：不需要序列化/反序列化
3. **即插即用**：不需要修改接收模型架构
4. **可扩展**：新增模型只需训练自己的编解码器

## 11.6 Vision Latent MAS OCR（OCR 基线）

### 11.6.1 核心思想

**位置**：`methods/vision_latent_mas_ocr.py`

```python
"""
OCR-image multi-agent communication method.

This is a *drop-in* method intended for the same evaluation harness as other methods.
It renders inter-agent notes as images and lets the receiver VLM read them
through the vision pathway (OCR-like behavior in the model).

So: agents do not exchange text tokens; they exchange an OCR note image.
The receiver ingests it through its vision encoder.
"""
```

### 11.6.2 通信流程

```
Agent A:
  生成文本 "我的计划是..."
      │
      ▼
  render_as_image("我的计划是...") → PIL Image
      │
      │ 传输图像
      ▼
Agent B:
  接收图像，通过视觉通道处理
  VLM B 执行 OCR 读取图像内容
      │
      ▼
  继续推理...
```

### 11.6.3 图像渲染

```python
# vision_latent_mas_ocr.py:...
def render_note_as_image(text: str, ...) -> Image.Image:
    """将文本渲染为图像"""
    img = Image.new('RGB', (width, height), color='white')
    draw = ImageDraw.Draw(img)

    # 渲染文本
    draw.text((margin, margin), text, fill='black', font=font)

    return img
```

### 11.6.4 特点

| 方面 | 说明 |
|------|------|
| **通信介质** | OCR 图像（通过视觉通道） |
| **信息损失** | 中等（文本→图像→OCR） |
| **延迟** | 高（需要渲染+OCR） |
| **异构支持** | 好（使用视觉通道） |
| **实现复杂度** | 低（无需训练） |

### 11.6.5 与 Vision Wormhole 的区别

| 方面 | OCR | Vision Wormhole |
|------|-----|-----------------|
| **传输内容** | 渲染的文本图像 | 连续潜在表示 |
| **信息形式** | 离散（图像化文本） | 连续（视觉特征） |
| **信息损失** | 高（文本→图像→OCR） | 低（直接潜在传递） |
| **需要训练** | 否 | 是 |

## 11.8 方法对比总结

### 11.8.1 功能对比表

| 方法 | 通信介质 | 异构支持 | 需要训练 | 信息损失 | 延迟 |
|------|----------|----------|----------|----------|------|
| Baseline | 无 | N/A | 否 | N/A | 无 |

### 11.8.2 适用场景

| 方法 | 最佳场景 |

## 11.10 本课小结

### 关键要点

1. **Baseline**：单模型基线，无通信开销
2. **Text MAS**：文本通信，实现简单但信息损失大
3. **Latent MAS**：潜在空间转移，需要对齐矩阵
4. **Vision Wormhole**：视觉通道通信，完全异构支持
5. **OCR**：图像渲染+OCR，快速原型方法
6. **Hybrid**：混合方法，平衡性能和多模态

### 思考题

1. 为什么 Text MAS 虽然"信息损失大"，但仍然是重要的基线？
2. Vision Wormhole 和 OCR 都使用视觉通道，它们在信息传递效率上的根本差异是什么？
3. 如果一个系统同时支持 Vision Wormhole 和 Text MAS，如何判断在什么场景下应该切换到文本通信（用于调试/日志）？
