# 第 13 课：教师-学生蒸馏目标

## 课程目标

- 理解视觉编解码器的训练目标

## 13.1 训练概述

### 13.1.1 教师-学生架构

Vision Wormhole 的编解码器使用**教师-学生蒸馏**进行训练：

```
教师 (Teacher):        学生 (Student):
┌─────────────────┐   ┌─────────────────┐
│  原始 VLM        │   │  编码器 → 解码器 │
│  (作为目标)      │   │  (需要训练)      │
└─────────────────┘   └─────────────────┘
        │                      │
        │  ground truth         │ 学习目标
        ▼                      ▼
   隐藏状态 h_T           重建隐藏状态 h_S
   Logits T               Logits S
```

### 13.1.2 训练目标

让学生的输出尽可能接近教师的输出，从而学会：
- **编码器**：将任意 VLM 的隐藏状态压缩到通用空间
- **解码器**：从通用空间重建适合目标模型的视觉 token

---

## 13.2 损失函数总览

### 13.2.1 四种损失

训练编解码器使用**多目标损失**：

| 损失 | 权重参数 | 作用 |
|------|----------|------|
| **MSE Loss** | `loss_mse_w` | 隐藏状态重建 |
| **KL Loss** | `loss_kl_w` | 下一个 token 预测对齐 |
| **Stats Loss** | `loss_stats_w` | 注入统计匹配 |
| **CKA Loss** | (可能不存在) | 隐藏状态相似性 |

### 13.2.2 损失计算代码

**位置**：`train_vision_latent_mas_codec_new.py:1691-1713`

```python
loss = torch.zeros((), device=device, dtype=torch.float32)
loss_mse = torch.zeros((), device=device, dtype=torch.float32)
loss_kl = torch.zeros((), device=device, dtype=torch.float32)
loss_stats = torch.zeros((), device=device, dtype=torch.float32)

# MSE 损失
if loss_mse_w > 0:
    loss_mse = F.mse_loss(student_h, teacher_h)
    loss = loss + loss_mse_w * loss_mse

# KL 损失
if loss_kl_w > 0:
    loss_kl = _compute_kl_loss(
        student_logits=student_logits,
        teacher_logits=teacher_logits,
        temp=float(cfg.vision_codec_kl_temp),
        mode=kl_mode,
        logit_clip=kl_logit_clip,
        topk=kl_topk,
    )
    loss = loss + loss_kl_w * loss_kl

# 统计损失
if loss_stats_w > 0:
    inj_rms = inj.float().pow(2).mean().sqrt().clamp_min(1e-6)
    loss_stats = F.mse_loss(inj_rms, dummy_rms.expand_as(inj_rms))
    loss = loss + loss_stats_w * loss_stats
```

## 13.3 MSE 损失 (Mean Squared Error)

### 13.3.1 原理

MSE 损失衡量**隐藏状态重建**的误差：

$$\mathcal{L}_{MSE} = \| h_{teacher} - h_{student} \|^2_2$$

其中：
- $h_{teacher}$：教师模型的隐藏状态
- $h_{student}$：学生（解码器）输出的隐藏状态

### 13.3.2 代码实现

```python
loss_mse = F.mse_loss(student_h, teacher_h)
```

### 13.3.3 作用

| 方面 | 说明 |
|------|------|
| **隐藏状态对齐** | 确保解码器输出的隐藏状态接近教师 |
| **粗粒度重建** | 捕捉整体激活模式 |
| **稳定性** | 梯度平滑，训练稳定 |

### 13.3.4 局限性

MSE 只能捕捉**二阶统计量**（均值和方差），对于分布的其他特征无能为力。

## 13.4 KL 散度损失

### 13.4.1 原理

KL 散度衡量两个概率分布的差异：

$$\mathcal{L}_{KL} = D_{KL}(P_T \| P_S) = \sum_i P_T(i) \log \frac{P_T(i)}{P_S(i)}$$

其中：
- $P_T$：教师模型的下一个 token 分布
- $P_S$：学生模型的下一个 token 分布

### 13.4.2 温度调节

使用**温度 (Temperature)** 调节分布的平滑度：

$$P_T^T = \text{softmax}\left(\frac{\log P_T}{T}\right)$$

```python
# train_vision_latent_mas_codec_new.py:219-221
T = max(1e-6, float(temp))
log_tgt = F.log_softmax(t_logits / T, dim=-1)
logp = F.log_softmax(s_logits / T, dim=-1)
```

| 温度 | 效果 |
|------|------|
| T < 1 | 分布更尖锐，强调主要 token |
| T = 1 | 原始分布 |
| T > 1 | 分布更平滑，增加探索 |

### 13.4.3 代码实现

**位置**：`train_vision_latent_mas_codec_new.py:201-229`

```python
def _compute_kl_loss(
    *,
    student_logits: torch.Tensor,
    teacher_logits: torch.Tensor,
    temp: float,
    mode: str,
    logit_clip: float,
    topk: int,
) -> torch.Tensor:
    # 清理 logits
    t_logits = _sanitize_logits(teacher_logits, clip=logit_clip)
    s_logits = _sanitize_logits(student_logits, clip=logit_clip)

    # Top-k 过滤
    if topk > 0 and topk < int(t_logits.shape[-1]):
        idx = torch.topk(t_logits, k=int(topk), dim=-1).indices
        t_logits = torch.gather(t_logits, dim=-1, index=idx)
        s_logits = torch.gather(s_logits, dim=-1, index=idx)

    # 温度调节
    T = max(1e-6, float(temp))
    log_tgt = F.log_softmax(t_logits / T, dim=-1)
    logp = F.log_softmax(s_logits / T, dim=-1)

    # KL 散度计算
    mode_n = _normalize_kl_mode(mode)
    if mode_n == "full":
        loss = F.kl_div(logp, log_tgt, reduction="batchmean", log_target=True)
        return loss * (T * T)  # 温度补偿

    penalty = _kl_penalty(logp, log_tgt, mode_n)
    return penalty.mean() * (T * T)
```

### 13.4.4 作用

| 方面 | 说明 |
|------|------|
| **分布对齐** | 让学生预测分布接近教师 |
| **Token 预测** | 直接优化下一个 token 的预测 |
| **细节保留** | 捕捉 logits 的细微差异 |

## 13.5 统计损失 (Statistics Loss)

### 13.5.1 原理

统计损失确保**注入信号的强度**与**原始 dummy image** 的统计特性匹配：

$$\mathcal{L}_{stats} = \| RMS(inj) - RMS(dummy) \|^2$$

其中 $RMS(x) = \sqrt{\text{mean}(x^2)}$ 是**均方根 (Root Mean Square)**。

### 13.5.2 代码实现

```python
# train_vision_latent_mas_codec_new.py:1710-1712
if loss_stats_w > 0:
    inj_rms = inj.float().pow(2).mean().sqrt().clamp_min(1e-6)
    loss_stats = F.mse_loss(inj_rms, dummy_rms.expand_as(inj_rms))
    loss = loss + loss_stats_w * loss_stats
```

### 13.5.3 作用

| 方面 | 说明 |
|------|------|
| **尺度匹配** | 确保注入信号不会过大或过小 |
| **数值稳定** | 防止梯度爆炸 |
| **视觉一致性** | 保持与原始图像相似的统计特性 |

## 13.6 损失权重配置

### 13.6.1 超参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `loss_mse_w` | >0 | MSE 损失权重 |
| `loss_kl_w` | >0 | KL 损失权重 |
| `loss_stats_w` | >0 | 统计损失权重 |
| `vision_codec_kl_temp` | 1.0 | KL 温度 |
| `kl_logit_clip` | 0 | logits 裁剪范围 |
| `kl_topk` | 0 | Top-k 过滤 |

### 13.6.2 配置示例

```bash
python train_vision_latent_mas_codec_new.py \
  --loss_mse_w 1.0 \
  --loss_kl_w 0.5 \
  --loss_stats_w 0.1 \
  --vision_codec_kl_temp 1.0 \
  --kl_logit_clip 50.0
```

## 13.7 前向传播流程

### 13.7.1 学生前向

**位置**：`train_vision_latent_mas_codec_new.py:298-349`

```python
def _student_forward_last_token_only(
    *,
    causal_module: nn.Module,
    forward_kwargs: Dict[str, Any],
) -> Tuple[torch.Tensor, torch.Tensor]:
    # 获取 backbone 和 lm_head
    backbone = _get_causal_backbone(causal_module)
    lm_head = _get_causal_lm_head(causal_module)

    # Backbone 前向（不含 lm_head）
    bkw = dict(forward_kwargs)
    bkw["use_cache"] = False
    bkw["output_hidden_states"] = False
    out_b = _forward_with_supported_kwargs(backbone, bkw)

    # 获取最后一个 token 的隐藏状态
    student_h = hidden[:, -1, :]

    # LM head 计算 logits
    student_logits = lm_head(student_h)
    student_logits = _apply_logit_softcap_if_configured(student_logits, causal_module)

    return student_h, student_logits
```

### 13.7.2 完整训练步骤

```
1. 教师前向
   input_ids → Teacher VLM → h_teacher, logits_teacher

2. 学生编码
   h_teacher → Encoder → U (通用表示)

3. 学生解码
   U → Decoder → delta, gate

4. 注入
   dummy_tokens + gate * delta → injected_tokens

5. 学生前向
   injected_tokens → Student VLM → h_student, logits_student

6. 损失计算
   loss = loss_mse_w * MSE(h_student, h_teacher)
        + loss_kl_w * KL(logits_student, logits_teacher)
        + loss_stats_w * Stats(injected)

7. 反向传播
   loss.backward()
   optimizer.step()
```

## 13.8 损失平衡

### 13.8.1 多目标优化

同时优化三个目标：
- **MSE**：隐藏状态重建
- **KL**：下一个 token 预测
- **Stats**：注入强度匹配

### 13.8.2 权重调整策略

| 场景 | 建议权重 |
|------|----------|
| 隐藏状态精度优先 | `loss_mse_w = 1.0, loss_kl_w = 0.1` |
| Token 预测精度优先 | `loss_mse_w = 0.1, loss_kl_w = 1.0` |
| 平衡 | `loss_mse_w = 1.0, loss_kl_w = 1.0, loss_stats_w = 0.1` |

可以逐渐增加 KL 损失的权重：

```python
# 后期：逐渐增加 KL 权重
```

## 13.9 本课小结

### 关键要点

1. **教师-学生蒸馏**：用原始 VLM 作为教师，训练编解码器作为学生
2. **MSE 损失**：`||h_teacher - h_student||²`，隐藏状态重建
3. **KL 损失**：$D_{KL}(P_T || P_S)$，下一个 token 预测对齐
4. **温度调节**：通过温度 T 控制分布平滑度
5. **统计损失**：RMS 匹配，注入强度一致性
6. **多目标优化**：三个损失加权求和

### 思考题

1. 为什么需要 KL 损失而不只用 MSE？两者有什么互补性？
2. 如果温度 T 设置为非常高的值（如 100），会发生什么？
3. 统计损失只匹配 RMS（均方根），是否足够？能否设计更精细的统计匹配？
