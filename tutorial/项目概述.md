# 项目概述: The Vision Wormhole

这是一个**异构多智能体系统 (MAS) 的潜在空间通信框架**，利用视觉-语言模型 (VLM) 实现智能体之间的"视觉虫洞"通信。

## 核心思想

传统多智能体系统通过**离散文本**通信，速度慢且信息损失大。而该框架将不同模型的推理轨迹映射到**统一的视觉潜在空间**，然后直接注入到接收者的视觉通道中——就像在智能体之间建立一个"虫洞"。

## 主要组件

| 组件 | 位置 | 功能 |
|------|------|------|
| `LatentToUniversalEncoder` | `methods/vision_latent_mas_codec_new.py:551` | 将任意 VLM 的潜在状态压缩为 K 个通用 token |
| `UniversalToVisionDecoder` | `methods/vision_latent_mas_codec_new.py:616` | 将通用表示解码回视觉 token |
| `ModelWrapper` | `models.py` | VLM 加载与管理，处理 KV cache |

## 通信方法对比

| 方法 | 说明 |
|------|------|
| `text_mas` | 标准文本通信 |
| `latent_mas` | 潜在空间转移（无视觉编码器） |
| `vision_latent_mas_codec_new` | **核心方法** - 使用视觉编解码器通信 |
| `vision_latent_mas_ocr` | OCR 基线（文本转图像） |

## 技术栈

- **框架**: PyTorch, Hugging Face Transformers, vLLM, SGLang
- **支持的 VLM**: Qwen-VL, Gemma3-VL, LFM2.5-VL, InternVL, MiniCPMV, SmolVLM
- **数据集**: GSM8K, GPQA, ARC, AIME, MBPP+, HumanEval+, MedQA

## 关键创新: Hub-and-Spoke 拓扑

传统的 N 个模型需要 O(N²) 个翻译器，而这个框架只需 O(N) 个——所有模型对齐到一个**通用视觉编解码器**（hub），每个模型只需自己的编码器/解码器（spoke）。

## 主要入口

- `run.py` - 运行实验的主入口
- `train_vision_latent_mas_codec_new.py` - 训练视觉编解码器
- `merge_vision_codec_checkpoints.py` - 合并不同模型族的编解码器

## 工作流程

1. **训练编解码器**: `train_vision_latent_mas_codec_new.py`
2. **合并跨模型族的编解码器**: `merge_vision_codec_checkpoints.py`
3. **运行分区实验**: `launch_partition_run.sh`

## Agent 角色

| 角色 | 功能 |
|------|------|
| Planner | 创建分步解决计划 |
| Critic | 评估计划并提供反馈 |
| Refiner | 根据反馈改进计划 |
| Judger | 生成最终答案 |

## 训练目标

训练视觉编解码器使用**教师-学生蒸馏**目标：

- **MSE 损失**: 在 prompt 结束位置的损失
- **KL 损失**: 下一个 token 预测的 KL 散度
- **CKA 损失**: 隐藏状态对齐
- **统计损失**: 注入统计匹配
