# Vision Wormhole 课程大纲

本课程旨在帮助学习者从零开始，系统掌握 **Vision Wormhole** 项目的核心思想、架构设计、实现细节和实验方法。课程共 20 节课，内容循序渐进，覆盖理论基础、代码实现、训练流程和实验设计。

---

## 第一阶段：基础概念（第 1-4 课）

### 第 1 课：多智能体系统 (MAS) 概述
- 什么是多智能体系统
- LLM 驱动的 MAS 及其应用场景
- 传统文本通信的局限性：延迟、信息损失
- 本课程的目标：实现高效异构通信

### 第 2 课：视觉-语言模型 (VLM) 基础
- VLM 的基本架构：视觉编码器 + 语言解码器
- 常见的 VLM 模型：Qwen-VL、Gemma3-VL、LFM2.5-VL、InternVL、SmolVLM
- 多模态推理轨迹的表示方式
- 项目支持的 VLM 列表及模型识别 (`models.py`)

### 第 3 课：潜在空间通信 (Latent Space Transfer)
- 为什么选择潜在空间而非文本？
- 连续表示 vs 离散表示
- 现有方法的局限性：同构架构依赖、成对翻译器
- Hub-and-Spoke 拓扑的优势：O(N²) → O(N)

### 第 4 课：Vision Wormhole 核心思想
- 将视觉接口作为"通用端口"
- Universal Visual Codec 的设计哲学
- 跨模型的文本无关 (text-free) 通信
- 虫洞隐喻：直接注入视觉通道

## 第二阶段：核心架构（第 5-8 课）

### 第 5 课：LatentToUniversalEncoder
- 输入：任意 VLM 的潜在状态 [B, L, H]
- Query-based 注意力机制压缩 L 个 token → K 个通用 token
- 三类 Query：语义 Query、全局 Query、风格 Query
- 风格信息注入：mean、std、norm 统计量
- 代码位置：`methods/vision_latent_mas_codec_new.py:551`

### 第 6 课：UniversalToVisionDecoder
- 输入：通用表示 [B, K, D]
- 可学习的 Query Tokens 生成 K_img 个视觉 token
- 门控机制 (Gating Mechanism)：控制注入强度
- 输出：delta [B, K_img, H_out] 和 gate [B, 1, 1]
- 代码位置：`methods/vision_latent_mas_codec_new.py:616`

### 第 7 课：_ResamplerBlock 与注意力机制
- MultiheadAttention 的使用
- LayerNorm 布局：ln_q、ln_kv 分离
- 前馈网络 (FFN)：4*dim 扩展 + GELU
- 残差连接的实现

### 第 8 课：编解码器的协同工作流程
- Sender 端：LatentToUniversalEncoder 将推理轨迹编码为通用表示
- 传输：跨模型的通用潜在空间
- Receiver 端：UniversalToVisionDecoder 将通用表示解码为视觉 token
- 注入：残差连接 + 门控的视觉通道注入

## 第三阶段：模型封装与多智能体实现（第 9-12 课）

### 第 9 课：ModelWrapper 设计与实现
- VLM 加载与管理 (`models.py`)
- KV Cache 操作：`_cache_to_legacy_list`、`_past_length`
- `generate_latent_batch`：潜在向量生成
- 多模态模型检测：`_is_internvl_config`、`_is_smolvlm_config`
- vLLM 集成支持

### 第 10 课：Agent 角色定义
- 四种角色：Planner、Critic、Refiner、Judger
- `prompts.py` 中的提示词模板
- 层次化 vs 顺序式消息构建
- `build_agent_message_hierarchical_latent_mas` vs `build_agent_message_sequential_latent_mas`

### 第 11 课：多智能体方法对比
- `text_mas`：标准文本通信基线
- `latent_mas`：纯潜在空间转移
- `vision_latent_mas_codec_new`：核心 Vision Wormhole 方法
- `vision_latent_mas_ocr`：OCR 基线（文本→图像）
- `latent_mas_hybrid`：混合方法
- `baseline`：单模型基线

### 第 12 课：Dummy Image 与视觉占位符
- 为什么需要虚拟图像？
- `_DummyImageSpec`：count 和 size 配置
- 视觉编码器的占位符注入
- 不同的 dummy image 策略

## 第四阶段：训练流程（第 13-16 课）

### 第 13 课：教师-学生蒸馏目标
- MSE 损失：prompt 结束位置的预测误差
- KL 损失：下一个 token 预测的分布对齐
- CKA 损失：隐藏状态相似性分析
- 统计损失：注入统计量的匹配
- `train_vision_latent_mas_codec_new.py` 训练脚本解析

### 第 14 课：训练超参数与配置
- `vision_codec_dim`：通用表示维度
- `vision_codec_tokens`：通用 token 数量
- `vision_codec_img_tokens`：图像 token 数量
- `vision_codec_heads`：注意力头数
- `vision_codec_layers`：Resampler 层数
- `latent_steps`：潜在向量生成步数
- 学习率、batch size、dropout 配置

### 第 15 课：跨模型族的编解码器合并
- 问题：不同模型族的编解码器如何统一？
- Hub-and-Spoke 合并策略
- `merge_vision_codec_checkpoints.py` 脚本解析
- Anchor Text 对齐机制
- 合并后的检查点格式

### 第 16 课：Anchor Text 与数据预处理
- 锚点文本的作用
- `scripts/preprocess_dataset.py` 数据预处理
- 支持的数据集：PRM800k、CoSE、OpenCodeReasoning
- JSONL 格式输出
