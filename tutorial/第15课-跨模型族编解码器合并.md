# 第 15 课：跨模型族的编解码器合并

## 课程目标

- 理解为什么需要合并编解码器
- 掌握 Hub-and-Spoke 合并策略
- 理解 Anchor Text 对齐机制
- 掌握合并脚本的使用方法

---

## 15.1 问题背景

### 15.1.1 单模型训练的问题

每个 VLM 都需要训练自己的编解码器：

```
训练阶段：
  Model A → 训练 Encoder_A + Decoder_A
  Model B → 训练 Encoder_B + Decoder_B
  Model C → 训练 Encoder_C + Decoder_C
```

### 15.1.2 通信问题

如果只有各自的编解码器：

```
A → B 通信:
  A 使用 Encoder_A
  B 使用 Decoder_B
  问题：Encoder_A 和 Decoder_B 没有对齐！

A → A 通信（自身）:
  A 使用 Encoder_A
  A 使用 Decoder_A
  OK：自身是对齐的
```

### 15.1.3 解决方案

合并所有模型的编解码器到**统一的通用空间**，使得：

```
任意模型之间的通信都经过同一个通用空间
  A → B: Encoder_A → Universal → Decoder_B
  A → C: Encoder_A → Universal → Decoder_C
  B → C: Encoder_B → Universal → Decoder_C
```

## 15.2 Hub-and-Spoke 合并策略

### 15.2.1 策略思想

**Hub-and-Spoke（中心-辐射）** 拓扑：

```
                    ┌─────────────────┐
                    │   Universal      │
                    │   Space (Hub)   │
                    └────────┬────────┘
                             │
         ┌───────────────────┼───────────────────┐
         │                   │                   │
         ▼                   ▼                   ▼
   ┌──────────┐       ┌──────────┐       ┌──────────┐
   │ Encoder A│       │ Encoder B│       │ Encoder C│
   │ Decoder A│       │ Decoder B│       │ Decoder C│
   └────┬─────┘       └────┬─────┘       └────┬─────┘
        │                   │                   │
        ▼                   ▼                   ▼
   ┌──────────┐       ┌──────────┐       ┌──────────┐
   │ Model A  │       │ Model B  │       │ Model C  │
   │ (Qwen)   │       │ (Gemma)  │       │ (LFM)    │
   └──────────┘       └──────────┘       └──────────┘
```

### 15.2.2 合并目标

让所有 Encoder 的输出和所有 Decoder 的输入都在**同一个通用空间**中：

- 所有 Encoder 的输出 $U_A, U_B, U_C$ 应该在同一个流形上
- 所有 Decoder 的输入应该能接受这个流形上的点

### 15.2.3 复杂度对比

| 方案 | 需要对齐的数量 | 复杂度 |
|------|----------------|--------|
| 成对对齐 | $N(N-1)/2$ | $O(N^2)$ |
| Hub-and-Spoke | $2N$ | $O(N)$ |

## 15.3 Anchor Text 对齐机制

### 15.3.1 核心思想

使用**锚点文本 (Anchor Text)** 作为对齐的桥梁：

```
锚点文本: "1+1=2", "2+2=4", "猫是动物", ...

  Model A 对锚点文本的表示: U_A(anchor)
  Model B 对锚点文本的表示: U_B(anchor)
  Model C 对锚点文本的表示: U_C(anchor)

目标: 找到一个变换 W，使得:
  U_B(anchor) ≈ W @ U_A(anchor)
  U_C(anchor) ≈ W @ U_A(anchor)
  ...
```

### 15.3.2 为什么锚点文本有效？

1. **语义覆盖**：锚点文本覆盖各种语义类型
2. **分布代表**：锚点文本的分布代表了模型的典型输入
3. **可获取**：锚点文本可以从标准数据集构建

### 15.3.3 对齐算法

对于参考模型 $M_{ref}$ 和目标模型 $M_i$：

```python
# 1. 获取锚点文本的通用表示
U_ref = encoder_ref(hidden_states_ref)  # [N, K, D]
U_i = encoder_i(hidden_states_i)        # [N, K, D]

# 2. 求解最优对齐矩阵 W
# 目标: min ||U_ref - W @ U_i||²
# 解析解: W* = (U_ref.T @ U_i)^-1 @ U_ref.T @ U_i
#      或使用岭回归正则化
```

## 15.6 对齐数学

### 15.6.1 问题形式化

给定：
- $U_{ref} \in \mathbb{R}^{N \times K \times D}$：参考模型的通用表示
- $U_i \in \mathbb{R}^{N \times K \times D}$：目标模型的通用表示

求对齐矩阵 $W \in \mathbb{R}^{D \times D}$，使得：

$$\min_W \| U_{ref} - W \cdot U_i \|_F^2$$

### 15.6.2 解析解

展平为矩阵形式：

$$\min_W \| U_{ref} - W U_i \|_F^2$$

对 $W$ 求导并设为零：

$$W^* = U_{ref} U_i^T (U_i U_i^T)^{-1}$$

### 15.6.3 岭回归正则化

为数值稳定性添加正则化：

$$W^* = U_{ref} U_i^T (U_i U_i^T + \lambda I)^{-1}$$

## 15.9 常见问题与解决

### 15.9.1 架构不匹配

```
Error: Codec architecture mismatch
```

**解决**：确保所有编解码器使用相同的架构参数训练：
- `vision_codec_dim`
- `vision_codec_tokens`
- `vision_codec_img_tokens`
- `vision_codec_heads`
- `vision_codec_layers`

### 15.9.2 锚点文本不足

**解决**：使用更多的锚点文本，或调整 `--vision_codec_align_max_anchors`

### 15.9.3 合并后性能下降

可能原因：
1. 锚点文本不够代表
2. 模型之间差异太大

**解决**：
1. 增加锚点文本数量
2. 使用更通用的锚点文本
3. 调整正则化参数 `--vision_codec_ridge`

## 15.10 本课小结

### 关键要点

1. **合并目的**：让所有模型共享同一个通用空间
2. **Hub-and-Spoke**：所有 Encoder/Decoder 对齐到统一空间
3. **Anchor Text**：使用锚点文本作为对齐的桥梁
4. **对齐算法**：岭回归求解 $W^* = U_{ref} U_i^T (U_i U_i^T + \lambda I)^{-1}$
5. **合并输出**：包含所有模型的 encoders/decoders 和对齐矩阵
