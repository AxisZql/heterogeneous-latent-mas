### 8.2.2 编解码器创建

**位置**：`methods/vision_latent_mas_codec_new.py:1233-1281`

```python
def _init_codec_modules_and_load(self, ckpt: Dict[str, Any]) -> None:
    enc_sd = ckpt.get("encoders", {})
    dec_sd = ckpt.get("decoders", {})

    for idx, wrapper in enumerate(self.models):
        H = _infer_hidden_size(wrapper)  # 获取隐藏维度
        device = wrapper.model.device

        # 创建编码器
        enc = LatentToUniversalEncoder(
            h_in=H,
            d_univ=self.codec_dim,      # 通用空间维度
            k_univ=self.codec_tokens,     # 通用 token 数量
            n_heads=self.codec_heads,
            n_layers=self.codec_layers,
            dropout=self.codec_dropout,
        ).to(device=device, dtype=torch.float32)

        # 创建解码器
        dec = UniversalToVisionDecoder(
            d_univ=self.codec_dim,
            h_out=H,                      # 目标模型隐藏维度
            k_img=self.codec_img_tokens,   # 图像 token 数量
            n_heads=self.codec_heads,
            n_layers=self.codec_layers,
            dropout=self.codec_dropout,
            gate_init_bias=self.codec_gate_init_bias,
        ).to(device=device, dtype=torch.float32)

        # 加载预训练权重
        key_name = wrapper.model_name
        if key_name in enc_sd and key_name in dec_sd:
            enc.load_state_dict(enc_sd[key_name], strict=True)
            dec.load_state_dict(dec_sd[key_name], strict=True)

        enc.eval()
        dec.eval()
        self.encoders.append(enc)
        self.decoders.append(dec)
```

## 8.3 完整通信流程

### 8.3.1 端到端架构图

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Agent A (Sender)                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  问题: "计算 15 + 27 = ?"                                          │
│                                                                      │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │  Model A (VLM) - 推理并生成文本响应                          │   │
│  │                                                               │   │
│  │  隐藏状态序列: h_A = [h_1, h_2, ..., h_L]                    │   │
│  │                     [B, L, H_A]                               │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                              │                                       │
│                              ▼                                       │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │  Encoder A: LatentToUniversalEncoder                         │   │
│  │                                                               │   │
│  │  输入: h_A [B, L, H_A]                                       │   │
│  │  输出: U [B, K+2, d_univ]                                    │   │
│  │                                                               │   │
│  │  U = Encoder(h_A)                                            │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                              │                                       │
└──────────────────────────────│───────────────────────────────────────┘
                               │
                               │ U (通用表示)
                               │ 跨模型传输
                               ▼
┌──────────────────────────────│───────────────────────────────────────┐
│                        传输层                                        │
│                    (共享内存/网络)                                    │
└──────────────────────────────│───────────────────────────────────────┘
                               │
                               ▼
┌──────────────────────────────│───────────────────────────────────────┐
│                    Agent B (Receiver)                                │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │  Decoder B: UniversalToVisionDecoder                         │   │
│  │                                                               │   │
│  │  输入: U [B, K+2, d_univ]                                    │   │
│  │  输出: delta [B, K_img, H_B], gate [B, 1, 1]                 │   │
│  │                                                               │   │
│  │  delta, gate = Decoder(U)                                    │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                              │                                       │
│                              ▼                                       │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │  虚拟图像准备与注入                                            │   │
│  │                                                               │   │
│  │  dummy_image_tokens = 虚拟图像的视觉 token                     │   │
│  │  injected_tokens = dummy_image_tokens + gate * delta          │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                              │                                       │
│                              ▼                                       │
│  ┌──────────────────────────────────────────────────────────────┐   │
│  │  Model B (VLM) - 接收注入的视觉 token 并继续推理               │   │
│  │                                                               │   │
│  │  输入: [文本 tokens + 注入的视觉 tokens]                       │   │
│  │  Model B 以为在看图像，实际接收了 Agent A 的潜在推理信息        │   │
│  └──────────────────────────────────────────────────────────────┘   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 8.4 维度转换追踪

### 8.4.1 完整维度变化

| 阶段 | 变量 | 形状 | 说明 |
|------|------|------|------|
| **Sender 推理** | `h_A` | $[B, L, H_A]$ | LLM 的隐藏状态序列 |
| **Encoder 输出** | `U` | $[B, K+2, d_{univ}]$ | 通用表示 |
| **Decoder 输出** | `delta` | $[B, K_{img}, H_B]$ | 视觉 token 增量 |
| **Decoder 输出** | `gate` | $[B, 1, 1]$ | 注入强度 |
| **注入后** | `vision_tokens` | $[B, K_{img}, H_B]$ | 增强的视觉 token |

### 8.4.2 典型配置值

| 参数 | 典型值 | 说明 |
|------|--------|------|
| `B` | 1-32 | Batch size |
| `L` | 128-4096 | 输入序列长度 |
| `H_A`, `H_B` | 2048-8192 | 模型隐藏维度 |
| `K` (codec_tokens) | 1024 | 通用 token 数量 |
| `K_img` (codec_img_tokens) | 256 | 视觉 token 数量 |
| `d_univ` (codec_dim) | 512 | 通用空间维度 |

### 8.5.3 为什么需要虚拟图像？

虚拟图像在系统中扮演重要角色：

| 角色 | 说明 |
|------|------|
| **占位符** | 占据视觉 token 槽位 |
| **载体** | 其视觉 token 位置可被 delta 替换 |
| **兼容性** | 不改变 VLM 的处理流程 |
| **可控性** | 可以精确控制注入位置和强度 |

## 8.6 视觉通道注入机制

### 8.6.1 注入公式

```python
# 注入操作
injected_tokens = dummy_image_tokens + gate * delta
```

### 8.6.2 注入图解

```
dummy_image_tokens:          delta (来自 Decoder):
┌────────────────────┐      ┌────────────────────┐
│ t_1  t_2  ... t_K  │      │ d_1  d_2  ... d_K  │
│  ↓    ↓       ↓    │      │  ↑    ↑       ↑    │
│[img] [img] [img]   │      │[inject info]        │
└────────────────────┘      └────────────────────┘
         │                           │
         │           ┌───────────────┘
         │           │ gate * delta
         ▼           ▼
┌────────────────────┐
│ t_1+  t_2+ ... t_K+│
│ ≈img   ≈img  ≈img  │ ← 增强后的视觉 token
└────────────────────┘
         │
         ▼
    输入到 VLM 的视觉通道
```

### 8.6.3 注入效果

| gate 值 | 效果 |
|---------|------|
| 0.0 | `injected = dummy`（无注入） |
| 0.5 | `injected = dummy + 0.5*delta` |
| 1.0 | `injected = dummy + delta` |
| >1.0 | 过度注入（通常不鼓励） |

```
          ┌──────────────────┼──────────────────┐

         ▼                  ▼                  ▼
    ┌──────────┐      ┌──────────┐      ┌──────────┐
    │  Model   │      │  Model   │      │  Model   │
    │    A     │      │    B     │      │    C     │

    └──────────┘      └──────────┘      └──────────┘
```

### 8.7.2 通信示例

**A → B 通信 (Qwen-VL → Gemma)**：

```python
# 1. A 使用自己的编码器
U = encoders[0](h_A)  # h_A [B, L, H_A] → U [B, K+2, d_univ]

# 2. U 传输到 B

# 3. B 使用自己的解码器
delta, gate = decoders[1](U)  # U → delta [B, K_img, H_B]

# 4. B 注入
injected = dummy_img_tokens[1] + gate * delta
```


## 8.8 对齐矩阵 (Alignment Matrix)

### 8.8.1 为什么需要对齐？

即使通过编解码器，某些情况下仍需要额外的对齐：

```python
# align_out_W, align_out_b: 输出对齐
h_aligned = h @ align_out_W + align_out_b

# align_in_W, align_in_b: 输入对齐
```

### 8.8.2 对齐矩阵的来源

```python
def _load_alignment_from_ckpt(self, ckpt: Dict[str, Any]) -> None:
    align = ckpt.get("align", None)
    out_map = align.get("out", {})
    in_map = align.get("in", {})

    for i, wrapper in enumerate(self.models):
        oi = out_map.get(key_name, out_map.get(str(i)))
        if isinstance(oi, dict) and "W" in oi and "b" in oi:
            self.align_out_W[i] = oi["W"].detach().float().cpu()
            self.align_out_b[i] = oi["b"].detach().float().cpu()
```

## 8.9 代码示例：简化通信流程

### 8.9.1 模拟 Sender 端

```python
# 假设这是 Agent A 处理一个问题
def sender_process(model_a, encoder_a, question):
    # 1. 模型推理
    response_a = model_a.generate(question)

    # 2. 获取隐藏状态序列
    h_a = model_a.get_hidden_states()  # [B, L, H_A]

    # 3. 编码到通用空间
    U = encoder_a(h_a)  # [B, K+2, d_univ]

    return U
```

### 8.9.2 模拟 Receiver 端

```python
# 假设这是 Agent B 接收信息
def receiver_process(model_b, decoder_b, U):
    # 1. 从通用空间解码
    delta, gate = decoder_b(U)
    # delta: [B, K_img, H_B]
    # gate: [B, 1, 1]

    # 2. 准备虚拟图像 token
    dummy_tokens = model_b.get_dummy_image_tokens()  # [B, K_img, H_B]

    # 3. 注入
    injected_tokens = dummy_tokens + gate * delta

    # 4. 重新构建输入（文本 + 注入的视觉 token）
    new_input = model_b.build_multimodal_input(text_tokens, injected_tokens)

    # 5. 继续推理
    response_b = model_b.generate(new_input)

    return response_b
```

## 8.10 本课小结

### 关键要点

1. **VisionLatentMASMethodCODECNew** 整合了多个模型、编码器、解码器
2. **编解码器初始化**：从检查点加载权重，创建编码器和解码器
3. **完整流程**：隐藏状态 → 编码 → 传输 → 解码 → 注入 → 继续推理
4. **虚拟图像作用**：占位符 + 注入载体
5. **注入机制**：`injected = dummy + gate * delta`
6. **Hub-and-Spoke**：任意两个模型都可以通过各自的编解码器通信
