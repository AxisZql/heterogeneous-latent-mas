## 14.2 架构参数

### 14.2.1 vision_codec_dim

**通用空间的维度**

```bash
--vision_codec_dim 256
```

| 维度 | 说明 |
|------|------|
| 较小 (128-256) | 更高压缩率，可能丢失信息 |
| 中等 (256-512) | 平衡压缩率和质量 |
| 较大 (512+) | 更完整表示，但计算量大 |

**影响分析**：

```
vision_codec_dim = 256:
  输入: h_A [B, L, 4096]
  压缩: 4096 → 256 (16x 压缩)

vision_codec_dim = 512:
  输入: h_A [B, L, 4096]
  压缩: 4096 → 512 (8x 压缩)
```

### 14.2.2 vision_codec_tokens

**通用 token 的数量**

```bash
--vision_codec_tokens 1024
```

| 数量 | 说明 |
|------|------|
| 较少 (256-512) | 更紧凑表示，适合短推理 |
| 中等 (1024-2048) | 平衡信息保留和压缩 |
| 较多 (4096+) | 更完整表示，但传输量大 |

**与 Delta 的关系**：

```
K_univ = 1024:
  Encoder 输出: [B, K_univ + 2, d_univ] = [B, 1026, 256]

  Decoder 输入: [B, K_univ + 2, d_univ]

  Decoder 输出: [B, k_img, h_out]
```

### 14.2.3 vision_codec_img_tokens

**生成的图像 token 数量**

```bash
--vision_codec_img_tokens 256
```

| 数量 | 说明 |
|------|------|
| 较少 (128) | 与某些 VLM 的视觉 token 数量匹配 |
| 中等 (256) | 常用配置 |
| 较多 (512+) | 需要 VLM 支持更长的视觉序列 |

**必须与 VLM 的 dummy image token 数量匹配！**

### 14.2.4 vision_codec_heads

**多头注意力的头数**

```bash
--vision_codec_heads 8
```

| 头数 | 说明 |
|------|------|
| 较少 (4) | 计算快，表达能力较弱 |
| 标准 (8) | 平衡计算和表达 |
| 较多 (16+) | 表达强，计算量大 |

**每个头的维度** = `d_univ / n_heads`

```
d_univ = 256, n_heads = 8:
  每头维度 = 256 / 8 = 32
```

### 14.2.5 vision_codec_layers

**ResamplerBlock 的层数**

```bash
--vision_codec_layers 4
```

| 层数 | 说明 |
|------|------|
| 较少 (1-2) | 计算快，可能欠拟合 |
| 标准 (4-6) | 平衡性能和计算 |
| 较多 (8+) | 表达强，但可能过拟合 |

**计算量影响**：

```
2 层 vs 4 层 vs 6 层:
  计算量: 1x    2x    3x
```

### 14.2.6 vision_codec_dropout

**Dropout 概率**

```bash
--vision_codec_dropout 0.1
```

| 值 | 说明 |
|------|------|
| 0.0 | 无 dropout，训练稳定 |
| 0.1 | 常用默认值，防止过拟合 |
| 0.2+ | 强正则化，可能欠拟合 |

## 14.3 训练参数

### 14.3.1 训练步数

```bash
--vision_codec_train_steps 200
```

| 步数 | 适用场景 |
|------|----------|
| 100-200 | 快速实验 |
| 400-800 | 标准训练 |
| 1000+ | 充分收敛 |

### 14.3.2 Batch Size

```bash
--vision_codec_train_batch_size 8
```

| Batch Size | 说明 |
|------------|------|
| 较小 (1-2) | 显存友好，但梯度估计噪声大 |
| 中等 (4-8) | 平衡显存和梯度质量 |
| 较大 (16+) | 梯度估计准，但显存需求高 |

**显存计算**：

```
Batch Size = 8, codec_dim = 256:
  约需: 8-12 GB GPU 显存

Batch Size = 2, codec_dim = 512:
  约需: 6-10 GB GPU 显存
```

### 14.3.3 学习率

```bash
--vision_codec_train_lr 5e-4
```

| 学习率 | 说明 |
|--------|------|
| 较小 (1e-5 到 1e-4) | 训练稳定，但收敛慢 |
| 中等 (2e-4 到 5e-4) | 标准配置 |
| 较大 (1e-3) | 收敛快，但可能不稳定 |

**学习率调度建议**：

```python
# 常用调度策略
scheduler = {
    "warmup_steps": 10,      # 预热步数
    "total_steps": 200,      # 总步数
    "peak_lr": 5e-4,         # 峰值学习率
    "min_lr": 1e-5           # 最小学习率
}
```

### 14.3.4 精度

```bash
--codec_uses_bf16
```

| 精度 | 说明 |
|------|------|
| FP32 | 全精度，稳定但慢 |
| BF16 | 混合精度，推荐使用 |
| FP16 | 混合精度，可能不稳定 |

## 14.4 损失权重参数

### 14.4.1 损失权重

```bash
--vision_codec_loss_mse 1.0
--vision_codec_loss_kl 0.25
--vision_codec_loss_stats 0.1
```

| 参数 | 默认值 | 说明 |
|------|--------|------|

```bash
--vision_codec_kl_mode "full"

```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `vision_codec_kl_mode` | "full" | KL 损失模式 |
| `vision_codec_kl_temp` | 1.0 | KL 温度 |
| `vision_codec_kl_logit_clip` | 80.0 | logits 裁剪范围 |

## 14.5 注入参数

### 14.5.1 注入裁剪

```bash
--vision_codec_latent_clip 50.0
--vision_codec_inj_clip 20.0
```

| 参数 | 说明 | 作用 |
|------|------|------|
| `latent_clip` | 采样潜在向量的裁剪 | 防止 latent_steps 间的数值爆炸 |
| `inj_clip` | 注入 token 的裁剪 | 防止注入信号过大 |

### 14.5.2 门控初始化

```bash
--vision_codec_gate_init_bias -4.0
```

**初始门控值** = `sigmoid(-4.0)` ≈ 0.018

这确保训练初期只有 ~2% 的信息被注入，帮助稳定训练。

## 14.8 参数影响矩阵

### 14.8.1 压缩率 vs 质量

| 参数 | 压缩率 | 信息保留 | 计算量 |
|------|--------|----------|--------|
| `codec_dim` ↓ | ↑ | ↓ | ↓ |
| `codec_tokens` ↓ | ↑ | ↓ | ↓ |
| `codec_img_tokens` ↓ | ↑ | ↓ | ↓ |

### 14.8.2 训练稳定性

| 参数 | 训练稳定性 | 收敛速度 |
|------|------------|----------|
| `dropout` ↑ | ↑ | ↓ |
| `gate_init_bias` ↓ | ↑ | ↓ |
| `lr` ↓ | ↑ | ↓ |

### 14.8.3 推荐的参数组合

| 场景 | dim | tokens | img_tokens | layers | batch_size |
|------|-----|--------|-------------|--------|------------|
| **快速实验** | 128 | 256 | 128 | 2 | 8 |
| **标准训练** | 256-512 | 512-1024 | 256 | 4 | 4-8 |
| **高质量** | 512-1024 | 1024-2048 | 512 | 6-8 | 2-4 |
| **高压缩** | 64-128 | 128-256 | 64 | 2 | 16 |

## 14.9 本课小结

### 关键要点

1. **架构参数**：
   - `codec_dim`：通用空间维度（128-1024）
   - `codec_tokens`：通用 token 数量（256-2048）
   - `codec_img_tokens`：图像 token 数量（需匹配 VLM）
   - `codec_heads` 和 `codec_layers`：注意力头数和层数

2. **训练参数**：
   - `train_steps`：训练步数（100-1000+）
   - `train_batch_size`：批量大小（1-16）
   - `train_lr`：学习率（1e-5 到 1e-3）

3. **损失权重**：
   - MSE（1.0）、KL（0.25）、Stats（0.1）是常用默认值

4. **注入参数**：
   - `gate_init_bias = -4.0` 确保初始稳定

### 思考题

1. 如果 `codec_tokens` 设得太小（如 32），会发生什么？信息损失主要来自哪里？
2. 为什么 `vision_codec_img_tokens` 必须与 VLM 的 dummy token 数量匹配？如果不匹配会有什么问题？
3. `train_batch_size` 和 `train_steps` 如何权衡？能否用更大的 batch size 减少训练步数？
